# IIDX ScoreBoard - 핵심 인프라 구축 및 기술 스택 선정 이유

> 프론트엔드 기초 인프라 설계 및 라이브러리 선정 배경을 기록한 문서입니다. 포트폴리오 활용을 위해 기술적 의사결정 과정을 상세히 담고 있습니다.

## 1. 프로젝트 목표

IIDX ScoreBoard React 프론트엔드 애플리케이션의 기반이 되는 인프라를 구축하고, 확장성과 유지보수성을 고려한 기술 스택을 선정합니다.

## 2. 기술 스택 선정 이유 (Tech Stack Rationale)

핵심 라이브러리 선정 시 **경량성, 생산성, 그리고 프로젝트의 특수 기능(서열표 관리)**을 최우선으로 고려했습니다.

### 📦 전역 상태 관리: Zustand (`^4.5.0`)

- **역할**: 사용자 세션, 스코어 필터, 전역 UI 상태 관리
- **선정 이유**:
  - Redux에 비해 보일러플레이트가 압도적으로 적어 초기 개발 속도가 빠릅니다.
  - Flux 패턴을 따르면서도 훅(Hook) 기반으로 동작하여 React와의 궁합이 뛰어납니다.
  - Context API의 불필요한 리렌더링 문제를 쉽게 해결할 수 있습니다.

### 📦 드래그 앤 드롭: `@dnd-kit` 확장 패키지

- **역할**: 관리자용 서열표 에디터 (곡 이동 및 정렬)
- **선정 이유**:
  - **Admin 서열표 관리** 기능의 핵심인 '티어 간 곡 이동'을 구현하기 위해 선정했습니다.
  - `react-beautiful-dnd` 보다 가볍고, 모듈화가 잘 되어 있어 필요한 기능만 골라 쓸 수 있습니다.
  - 모바일 터치 환경과 키보드 접근성을 기본으로 지원하여 관리자 경험(AX)을 극대화합니다.

### 📦 사용자 피드백: `react-hot-toast` (`^2.4.0`)

- **역할**: 중요 액션에 대한 실시간 알림
- **선정 이유**:
  - "로그인 성공", "파일 업로드 완료" 등 사용자 경험(UX)에 필수적인 피드백을 직관적으로 제공합니다.
  - 프로미스(Promise) 기반 알림 처리가 매우 간편하여 비동기 작업 결과 표시가 용이합니다.

### 📦 스타일링 및 유틸리티

- **`clsx`**: Tailwind CSS 클래스를 조건부로 결합할 때 코드 가독성을 획기적으로 높여줍니다.
- **`date-fns`**: 백엔드의 ISO 날짜 데이터를 사용자 친화적인 형식(예: '방금 전', '2024.02.17')으로 변환하는 가벼운 유틸리티입니다.
- **`react-icons`**: 다양한 아이콘 셋을 하나로 통합하여 일관된 UI 요소를 빠르게 구현할 수 있게 합니다.

## 3. 폴더 구조 설계

관심사 분리(Separation of Concerns)를 위해 다음과 같은 모듈형 구조를 채택했습니다.

```
src/
├── components/
│   ├── common/          # 공용 UI 컴포넌트 (Atomic)
│   ├── layout/          # 프레임워크 (Header, Footer, Layout)
│   ├── auth/            # 인증 로직 전용 컴포넌트
│   ├── dashboard/       # 데이터 시각화 위젯
│   ├── scores/          # 스코어 관리 모듈
│   ├── tier-table/      # 서열표 뷰어
│   ├── admin/           # 관리자 도구 (핵심: DnD 에디터)
│   └── guards/          # 권한 및 라우트 보호
├── pages/               # 라우트별 페이지 진입점
├── hooks/               # 커스텀 훅 (비즈니스 로직 캡슐화)
├── store/               # 전역 상태 저장소 (Zustand)
├── utils/               # 순수 함수 및 유틸리티
└── data/                # 서열표 기준 데이터 (JSON)
```

## 4. 환경 변수 및 보안 설정

- **CSRF 보호**: Axios 인터셉터를 통해 쿠키에서 CSRF 토큰을 추출하고 헤더에 자동 포함하도록 설정했습니다.
- **환경 분리**: `.env.local`과 `.env.example`을 통해 개발 환경에 따른 API 엔드포인트를 유연하게 관리합니다.
