# v0.1.0 — 인프라 셋업 (2026-02-20)

## 개요

IIDX 점수 관리 서비스의 프론트엔드 MVP 기반을 구축하는 단계.  
백엔드(Spring Boot)는 이미 존재하므로, 클라이언트 단독 프로젝트로 분리 구성.

---

## 기술 스택 선택

### React + Vite

**왜 Vite인가?**

- Create React App(CRA)은 2023년 공식 유지보수 종료 → 새 프로젝트에서 사용 지양
- Vite는 ES Module 기반 번들링으로 HMR(Hot Module Replacement) 속도가 CRA 대비 10배 이상 빠름
- `@vitejs/plugin-react`로 React 지원이 1등급으로 제공됨

**왜 React인가?**

- 팀 친숙도 및 생태계 규모
- Zustand, react-router-dom 등 목표 스택이 React 기반

### Tailwind CSS v4

**왜 Tailwind인가?**

- 유틸리티 클래스 기반으로 디자인 결정이 마크업 안에 응집됨
- 별도 `.css` 파일을 최소화하여 컴포넌트 단위 개발에 적합
- v4는 PostCSS 없이 Vite 플러그인(`@tailwindcss/vite`)만으로 동작

**커스텀 색상 전략**  
IIDX 게임의 클리어 타입(FC, EX HARD, HARD, CLEAR, FAIL)은 게임 원본 배색을 따름.  
CSS 변수로 정의하여 다크모드 확장 가능성을 열어 둠.

```css
/* index.css */
:root {
  --color-fc: #ff6b6b; /* Full Combo - 핑크/레드 */
  --color-ex-hard: #ffd700; /* EX Hard - 골드 */
  --color-hard: #ffffff; /* Hard - 흰색 */
  --color-clear: #87ceeb; /* Clear - 하늘색 */
}
```

| `react-icons`      | ^5.x | 아이콘           | Fi(Feather) 계열로 일관된 선 스타일                               |

## 폴더 구조와 그 이유

```
src/
├── api/          # 서버와의 통신 함수 (axios 호출)
├── components/   # 재사용 가능한 UI 컴포넌트
│   ├── common/   # Button, Input, Spinner 등 범용 컴포넌트
│   ├── layout/   # Header, ProtectedLayout 등 페이지 골격
│   ├── auth/     # 인증 관련 전용 컴포넌트
│   ├── scores/   # 스코어 목록 전용 컴포넌트
│   ├── dashboard/# 대시보드 차트/카드 컴포넌트
│   ├── tier-table/# 서열표 컴포넌트
│   └── guards/   # 라우트 보호 컴포넌트
├── hooks/        # 데이터 페칭/상태 로직 훅 (useScores 등)
├── pages/        # 라우트에 직접 매핑되는 페이지 컴포넌트
├── store/        # Zustand 전역 상태
├── utils/        # 순수 함수 유틸리티
└── data/         # (정적) 상수, 목업 데이터
```

**왜 이 구조인가?**

- `pages/`와 `components/`를 분리하면 **페이지는 데이터 조합**, **컴포넌트는 표현**만 담당하는 역할 분리가 명확
- `hooks/`로 데이터 페칭 로직을 분리하면 컴포넌트가 UI에만 집중 가능 (테스트도 쉬워짐)
- `api/`를 별도 레이어로 두면, 서버 URL이나 파라미터가 바뀌어도 한 곳만 수정

## 환경 변수 설정

```
VITE_API_BASE_URL=http://localhost:8080
```

`VITE_` 접두사가 없으면 브라우저에서 접근 불가 (Vite 보안 정책).  
`.env.example`을 커밋하고 `.env.local`은 `.gitignore`에 추가하는 것이 표준.

## 배운 점 / 트러블슈팅

- **Tailwind v4 변경사항**: `tailwind.config.js` 파일이 더 이상 필요 없음. `@tailwindcss/vite` 플러그인이 자동 스캔
- **IIDX 테마 색상**: 게임 원본 UI를 분석하여 clear type별 배색 정의 → CSS 변수로 관리
