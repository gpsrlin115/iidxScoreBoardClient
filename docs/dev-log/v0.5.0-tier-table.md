# IIDX ScoreBoard: 서열표(Tier Table) 백엔드 영속화 및 연동 완료 보고서

## 1. 개요
프론트엔드의 `localStorage` 기반 서열표 데이터를 서버 데이터베이스(MariaDB)로 이관하여 중앙 영속화를 달성하고, 관리자 권한 기반의 데이터 관리 체계를 구축하였습니다.

---

## 2. 백엔드 구현 사항 (Spring Boot)

### 2.1 데이터베이스 및 엔티티
*   **Flyway 마이그레이션 (`V3`)**: `tier_table` (레벨/스타일별 JSON 저장) 생성 및 `users.role` 컬럼 추가.
*   **TierTable 엔티티**: `level`, `playStyle`, `tierData` (JSON 형식) 필드 매핑 및 `@Builder` 패턴 적용.
*   **Repository**: `TierTableRepository` (조회/저장) 및 `SongRepository` (서열표 미지정 곡 추출용 커스텀 쿼리) 구현.

### 2.2 보안 및 권한 제어
*   **메소드 보안 활성화**: `SecurityConfig`에 `@EnableMethodSecurity` 적용.
*   **접근 제어**: `TierTableController`에서 `@PreAuthorize("hasRole('ADMIN')")`를 통해 관리자 전용 저장/목록 조회 기능 보호.
*   **AuthService 수정**: 로그인/회원가입 시 사용자의 권한(`ROLE_USER`, `ROLE_ADMIN`)을 Spring Security 컨텍스트에 반영.
*   **DTO 및 서비스 개선**: `UserResponse` DTO에 `role`을 포함하게 하여 프론트엔드가 실시간으로 관리자 여부를 식별할 수 있도록 수정함.

### 2.3 API 엔드포인트
*   `GET /api/tiers/{level}/{playStyle}`: 서열표 조회 (모든 사용자)
*   `POST /api/tiers/{level}/{playStyle}`: 서열표 저장/갱신 (관리자 전용)
*   `GET /api/tiers/songs/{level}/{playStyle}`: 레벨/스타일별 전체 곡 제목 조회 (관리자 전용)

---

## 3. 프론트엔드 구현 사항 (React)

### 3.1 API 클라이언트 연동 (`src/api/tiers.js`)
*   **브라우저 의존성 제거**: `localStorage` 호출 로직을 제거하고 `apiClient`를 통한 실제 서버 통신 코드로 교체.
*   **호환성 매핑**: 백엔드의 곡 제목(`String`) 배열을 프론트엔드 스토어가 기대하는 `{ title: "..." }` 객체 배열로 자동 변환하여 연동.

### 3.2 관리자 스토어 연동 (`src/store/adminTierStore.js`)
*   데이터 로드 시 서버로부터 실시간 서열표 데이터와 관리 대상 곡 목록을 받아오도록 연동 완료.

---

## 4. 검증 결과
*   **권한 테스트**: 일반 사용자가 저장 API 호출 시 애플리케이션 레벨에서 차단(`AuthorizationDeniedException`)됨을 확인.
*   **데이터 정합성**: 서버 데이터 저장 후 새로고침 시에도 동일한 데이터가 유지됨을 확인.
*   **DTO 확인**: 로그인 응답에서 `role: "ADMIN"` 정보가 정확히 수신되도록 수정됨.

## 5. 관리자 가이드 (Ad-hoc)
특정 사용자(예: `sjkim1070`)에게 관리자 권한을 부여하려면 아래 Docker 명령어를 실행하십시오.
```bash
docker exec -it iidx-mariadb mariadb -u root -p<YOUR_ROOT_PASSWORD> -e "UPDATE iidx_score_board.users SET role = 'ADMIN' WHERE username = 'sjkim1070';"
```
